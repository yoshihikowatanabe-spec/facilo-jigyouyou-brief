新機能

クリエイティブの主軸となる2つの新機能

この2つの機能により、事業用クラウドは「事業用仲介ツールとして胸を張って言えるレベル」になる（向井さん談）。

NEW FEATURE 01

物件概要書 AI自動書き換え

物元（売主側仲介）から届いた物件概要書（PDF/Excel/画像）をアップロードするだけで、AIが40項目以上を自動で読み取り、自社の書式に変換して登録する機能。

解決するペイン
事業用仲介では、物元から届く概要書を「自社フォーマットに書き換える」作業が日常的に発生する。1件あたり数十分かかる手作業で、仕入れ担当の時間を大きく圧迫している。
単なる「取込」ではない
OCRで読み取って登録するだけでなく、自社の書式・項目体系に合わせて再構成するのがポイント。競合のレリーズ（レメティス社）はこの機能単体で導入が決まるケースがあるほど、事業用仲介のど真ん中のペイン。
訴求のキーワード
「書き換え」「自社書式に変換」「転記作業からの解放」「数十分→数秒」
NEW FEATURE 02

非構造化AIマッチング

営業の商談メモ・顧客ヒアリングメモなど、整理されていないテキストデータをAIが読み取り、物件情報と顧客ニーズを自動で照合する機能。

解決するペイン
事業用仲介では「あの物件を誰に出すか」が営業個人の記憶と人脈に依存している。ベテラン営業の頭の中にしかないマッチング判断を、AIが仕組み化する。
「共有」ではなく「マッチング」
この機能が解くのは「情報共有」の問題ではなく、「物件と顧客の照合」の問題。普段の商談メモを書くだけで、特別な入力・整理なしにAIがマッチングを行う。
訴求のキーワード
「記憶に頼らない」「メモを書くだけで」「提案漏れゼロ」「属人化の解消」
市場理解

事業用不動産の基本構造 — 居住用との違い

居住用（購入・売却・賃貸クラウド）との根本的な違いを整理する。

ワークフローの軸
居住用 =「顧客軸」（顧客 → 物件を探す）
事業用 =「物件軸」（物件 → 出す先を探す）
意思決定
居住用 = 感情・生活イメージ・家族合意
事業用 = 数字・事業計画・投資判断
情報流通
居住用 = SUUMO・レインズ等の集約PFあり
事業用 = 集約PFなし。物件概要書の直接流通
基本ドキュメント
居住用 = 販売図面（マイソク）
事業用 = 物件概要書（物元作成→仲介が自社書式に書き換え）

事業用仲介の核心課題：集約プラットフォームがないため、物件情報は各社がExcel・紙・メールで個別管理。「あの物件を誰に出すか」は営業個人の記憶と人脈に依存しており、組織的な物件マッチングの仕組みがない。これがFacilo事業用クラウドが解こうとしている問題。

ターゲット

詳細

セグメント別の特性と新機能の刺さり方

🏢

オフィス

賃貸が主流（一棟売買は投資に近い）。事業用で最もボリュームが大きい主戦場
顧客像
法人の総務・管理部門。移転・拡張・統合がトリガー。テナント代表は総務部長クラスが多い
取引の特徴
坪単価×面積で比較。共益費・フリーレント・原状回復など交渉変数が多い。3〜6ヶ月が標準
営業フロー
ニーズヒアリング → 複数ビルの概要書送付 → 内覧 → 条件交渉 → 申込。物件比較表を自社書式で作成が一般的
競合環境
三鬼商事・三幸エステート・CBRE等。ビル情報DBは各社独自構築で、それ自体が競争優位
新機能の刺さり方
概要書の書き換え頻度が最も高いセグメント → 自動書き換えの最直球ターゲット。「このニーズに合うビルは？」の照合 → AIマッチングも自然にハマる
🏪

店舗

賃貸（テナント出店）。飲食・小売・サービス業。意思決定が速く、感覚値も入る
顧客像
個人事業主・中小法人のオーナー。飲食店・美容室・クリニック等。FC本部がテナント探しをするケースも
取引の特徴
立地と人通りが全て。居抜き物件は特に動きが速い。1〜3ヶ月。鮮度が命
営業フロー
空き物件情報を出店希望者にスピード配信するのが勝負。内覧→即決もあり得る
競合環境
居抜き市場・テンポスマート等の専門PFがある程度機能。事業用の中では標準化が進んでいる方
新機能の刺さり方
件数が多くサイクルが短い → 出し忘れ・マッチング漏れが実害に直結。AIマッチングで提案速度を上げる訴求が有効。概要書は簡素なため書き換え訴求の優先度はやや低い
🏭

工場

賃貸・売買ともにあり。用途地域・環境規制が絡み、専門性が極めて高い
顧客像
製造業法人の経営層・工場長。移転は全社意思決定で取締役会マター
取引の特徴
スペック要求が極めて具体的：天井高・床耐荷重・電力容量・排水設備・用途地域。合致物件が少なく6〜12ヶ月+
営業フロー
要件定義 → リストアップ → 現地調査 → 法規制確認 → 契約。工場特化のベテラン仲介者が中心
情報の属人性
最も属人度が高い。「この工場はあの会社の要件に合う」は数値照合だけでは判断不能、経験則と土地勘に大きく依存
新機能の刺さり方
属人度が最も高い → AIマッチングの価値が最大のセグメント。概要書のスペック項目も40+と多く、自動書き換えの工数削減効果も大きい
📦

倉庫

賃貸が主流。EC拡大で需要増。物流施設との境界がグラデーション
顧客像
物流会社・EC事業者・小売の物流/購買部門。3PL事業者が借り手になるケースも多い
取引の特徴
IC距離・天井高・トラックバース数・荷捌きスペースが重要。立地は「配送効率」で評価
営業フロー
オペレーション要件ヒアリング → 候補提案 → 現地確認。「地図上で見る」ニーズが強い
市場トレンド
EC拡大で慢性的な供給不足。既存倉庫の転用・リノベーション案件が増加
新機能の刺さり方
スペック適合が比較的明確なためAIマッチングが機能しやすい。概要書の書き換えニーズもあり
🚛

物流施設

売買（開発・投資）+ テナントリーシング。プレイヤーが限定的で取引単価が大きい
顧客像
デベロッパー・物流大手・機関投資家・REIT。プレイヤーが限られている
取引の特徴
取引単価が数十億〜数百億円。土地取得→開発→リーシング→売却の多段階。財務モデルが意思決定の中心
営業フロー
1案件で複数の仲介フェーズが発生し、期間1年超が普通
情報の属人性
プレイヤー少数のため人脈=情報力。情報の非対称性そのものがビジネス
新機能の刺さり方
取引件数少×単価大で1件の出し忘れの機会損失が極めて大きい。概要書は高度に専門的で項目数も多く、自動書き換えの効果大
🏘

投資用一棟マンション

売買（投資目的の一棟購入）。利回り至上主義で居住用とは全く異なる
顧客像
富裕層個人・資産管理法人・不動産投資法人。「住む」ではなく「運用する」目的。利回り・キャッシュフローで判断
取引の特徴
表面利回り・稼働率・修繕履歴が判断材料。レントロール（賃料明細）が必須書類
営業フロー
投資基準ヒアリング → 物件情報配信 → 現地確認 → 融資打診 → 買付。スピード勝負で良い物件は数日で決まる
市場の特殊性
居住用マンションと物件形態は似るが動機・評価軸・顧客属性が全く異なる。投資用専門の仲介会社が棲み分け
新機能の刺さり方
「この投資家の基準に合う物件が出たら即提案」がニーズの核心。AIマッチングは「物件→投資家」方向でハマる。概要書書き換え（レントロール統合等）もニーズあり
補足

AIマッチングの「方向」はセグメントで逆転する

事業用は基本「物件軸」（物件→出す先を探す）だが、投資用一棟は「投資家軸」（投資家の基準→物件を探す）になる。AIマッチングの訴求はセグメントごとにどちら方向のマッチングかが変わる。

セグメント マッチングの方向 コピーの方向性
オフィス 双方向（テナント↔ビル） 「このニーズに合うビルは？」「このビルに興味を持ちそうなテナントは？」
店舗 物件 → 出店希望者 「空いたテナント、誰に出す？」
工場 物件 → 製造業法人 「この工場のスペックに合う会社は？」
倉庫 物件 → 物流企業 「この倉庫を必要としている会社は？」
物流施設 土地 → デベロッパー 「この用地に興味を持つデベは？」
投資用一棟 投資家 → 物件（逆方向） 「この投資家の基準に合う物件が出た」
